Abstract

This report documents the 10-week internship completed at Sajilo Digital Pvt. Ltd. as a Frontend Developer, focusing on the complete modernization and redesign of the Code for Change Nepal website. The primary objective was to migrate an existing static HTML-based website to a dynamic, component-based ReactJS application with improved user experience, performance, and maintainability.

The project involved analyzing the existing HTML structure, designing a new component architecture, implementing a responsive design system, and developing interactive features using ReactJS. Key technical implementations included creating reusable React components, implementing client-side routing with React Router, managing application state with React Hooks and Context API, optimizing performance through code splitting and lazy loading, and ensuring full responsiveness across all device types.

The redesigned website features a modern user interface, improved navigation structure, dynamic content rendering, interactive elements, and enhanced performance metrics. The migration from static HTML to ReactJS provided the organization with a more maintainable codebase, better development workflow, and foundation for future feature additions.

This internship provided comprehensive exposure to the process of modernizing legacy web applications, component-based architecture design, responsive web development, and performance optimization techniques using modern frontend technologies.

Chapter 1: Introduction

1.1 Introduction
In the rapidly evolving digital landscape, non-profit organizations like Code for Change Nepal require modern, engaging, and performant websites to effectively communicate their mission, showcase their work, and engage with volunteers and donors. This internship report details my work as a Frontend Developer at Sajilo Digital Pvt. Ltd., where I undertook the complete redesign and modernization of the Code for Change Nepal website from a static HTML-based platform to a dynamic ReactJS application.

The original website, built with traditional HTML, CSS, and JavaScript, faced limitations in maintainability, user engagement, and performance. The migration to ReactJS provided an opportunity to implement modern web development practices, including component-based architecture, efficient state management, and optimized performance. The redesigned platform aims to better represent the organization's mission of social change through technology while providing an improved user experience for visitors, volunteers, and donors.

This project provided practical experience in analyzing legacy codebases, designing modern architectures, and implementing full website redesigns using contemporary frontend technologies.

1.2 Problem Statement
The existing Code for Change Nepal website presented several challenges that necessitated a complete redesign:

- Outdated Technology Stack: The website was built with static HTML, making content updates difficult and requiring technical expertise for even minor changes.
- Poor Maintainability: Lack of component reusability led to code duplication and inconsistent implementations across pages.
- Limited Interactivity: Minimal JavaScript functionality resulted in a static user experience with limited engagement opportunities.
- Non-Responsive Design: The website did not adapt well to mobile devices, alienating a significant portion of potential visitors.
- Slow Performance: Unoptimized assets and inefficient code structure resulted in slow loading times and poor user experience.
- Inconsistent Design: Visual inconsistencies across pages undermined the organization's professional image.
- Difficult Content Management: Adding new projects, events, or updates required manual HTML editing, limiting the organization's ability to keep content current.
- Poor SEO Structure: The static nature limited opportunities for search engine optimization and content discovery.

1.3 Objectives
The primary objectives of this frontend redesign project were:

- To analyze the existing HTML-based website and identify areas for improvement in structure, design, and functionality.
- To design and implement a modern, component-based architecture using ReactJS for improved maintainability and scalability.
- To create a responsive design system that provides optimal user experience across all device types and screen sizes.
- To implement interactive features and dynamic content rendering to enhance user engagement.
- To optimize website performance through code splitting, asset optimization, and efficient rendering techniques.
- To improve the visual design and user interface while maintaining alignment with the organization's mission and branding.
- To implement a structured content management approach that allows for easier updates and additions.
- To enhance accessibility features ensuring the website is usable by people with disabilities.
- To implement proper SEO structures and metadata management for improved online visibility.

1.4 Scope and Limitations
Scope:
- Complete frontend redesign and development using ReactJS.
- Migration of all existing content from HTML to React components.
- Implementation of responsive design for mobile, tablet, and desktop devices.
- Development of reusable component library for consistent UI implementation.
- Integration of interactive features (image galleries, event calendars, donation forms).
- Performance optimization including image optimization, code splitting, and lazy loading.
- Implementation of client-side routing for seamless navigation.
- Basic SEO implementation with proper metadata and semantic HTML.
- Deployment to production hosting environment.

Limitations:
- Backend development for content management system was beyond scope; content remains statically managed.
- User authentication and membership features were not implemented in this phase.
- Advanced features like real-time chat, video streaming, or complex data visualizations were not included.
- The project focused on frontend modernization only; backend API integration was limited to mock data.
- Multilingual support was planned but not implemented in the initial release.
- Advanced analytics integration beyond basic Google Analytics was not included.

2.5 Literature Review / Related Study
The modernization of the Code for Change Nepal website was informed by contemporary frontend development practices:

- ReactJS and Component Architecture: Resources like "React Design Patterns and Best Practices" by Carlos Santana Roldán (2019) guided the implementation of reusable, maintainable component structures, emphasizing separation of concerns and single responsibility principle.

- Legacy Code Modernization: Research on "Modernizing Legacy Systems" by Robert C. Seacord (2003) informed the approach to analyzing existing HTML structures and developing migration strategies that minimize disruption while maximizing benefits.

- Responsive Web Design: Principles from Ethan Marcotte's foundational work (2011) were applied, utilizing fluid grids, flexible images, and CSS media queries to ensure accessibility across all screen sizes.

- Performance Optimization: Studies from Google's Web Vitals initiative and "High Performance Web Sites" by Steve Souders (2007) informed techniques for minimizing load times through asset optimization, code splitting, and efficient rendering.

- Accessibility Standards: WCAG 2.1 guidelines and "Inclusive Design Patterns" by Heydon Pickering (2016) influenced implementation decisions to ensure the website is usable by people with diverse abilities.

- SEO Best Practices: Current search engine optimization guidelines from Google's Webmaster Central informed implementation of semantic HTML, metadata, and site structure improvements.

- Testing and Quality Assurance: Conducting cross-browser testing, responsive testing, and accessibility testing.
- Deployment: Configuring and deploying the React application to production hosting environment.
