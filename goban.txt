Abstract

Ltd. as a Full Stack Developer, focusing on the design, development, and deployment of the Mount Glacier Trek adventure tourism website. The primary objective was to create a comprehensive online platform that facilitates trek bookings, provides detailed information about trekking packages, and manages customer interactions for a tourism business.

Chapter 1: Introduction

1.1 Introduction
The adventure tourism industry in Nepal has experienced significant digital transformation, with online platforms becoming essential for trekking companies to reach global audiences, manage bookings, and provide comprehensive information to potential clients. This internship report details my work as a Full Stack Developer at Sajilo Digital Pvt. Ltd., where I developed the Mount Glacier Trek website—a complete online solution for trekking package management and booking.

The project involved building a full-featured web application using the MERN stack (MongoDB, Express.js, React.js, Node.js) that serves both customers seeking trekking adventures and administrators managing the business operations. The platform was designed to handle the complete customer journey from browsing trek packages and checking availability to making secure bookings and payments.

This comprehensive development experience provided practical exposure to database design, REST API development, user authentication, payment gateway integration, and responsive frontend implementation—all essential components of modern full stack web development.

1.2 Problem Statement
Mount Glacier Trek, a growing adventure tourism company, faced several challenges in managing their operations and customer interactions:

- Manual Booking Processes: Reliance on phone/email for bookings led to errors, double bookings, and inefficient customer service.
- Limited Online Presence: Basic website provided only static information without interactive booking capabilities.
- Inefficient Customer Management: No centralized system to track customer inquiries, bookings, or payment status.
- Poor Information Accessibility: Potential clients couldn't easily browse available dates, compare packages, or check real-time availability.
- Administrative Overhead: Staff spent excessive time managing bookings manually rather than focusing on customer service and operations.
- No Payment Integration: Customers couldn't make secure online payments, requiring offline transactions.

This project aimed to address these challenges by developing a comprehensive digital platform that automates booking processes, provides real-time information, and streamlines business operations.

1.3 Objectives
The primary objectives of this full stack development project were:

- To design and develop a complete tourism management system with customer-facing frontend and administrative backend.
- To implement secure user authentication and authorization for both customers and administrators.
- To create a dynamic booking system with real-time availability checking and confirmation.
- To integrate secure payment gateway for online transactions.
- To develop an admin dashboard for managing treks, bookings, users, and content.
- To implement responsive design ensuring optimal user experience across all devices.
- To design scalable database architecture using MongoDB to handle trek data, user information, and booking records.
- To build RESTful APIs with Node.js and Express for seamless frontend-backend communication.
- To ensure application security through input validation, authentication middleware, and secure payment processing.

1.4 Scope and Limitations
Scope:
- Full stack development using MERN (MongoDB, Express.js, React.js, Node.js) technology stack.
- Frontend development with ReactJS including state management, routing, and responsive design.
- Backend development with Node.js and Express including REST API design and implementation.
- Database design and implementation with MongoDB including schema design and optimization.
- User authentication and authorization using JWT (JSON Web Tokens).
- Payment gateway integration for secure online transactions.
- Admin dashboard development for content and booking management.
- Deployment of both frontend and backend to production environment.

Limitations:
- Mobile application development was beyond the scope of this project.
- Advanced features like real-time chat support, GPS tracking, or social media integration were not implemented.
- The payment gateway was integrated in test mode; full production implementation requires additional compliance.
- Multi-language support and currency conversion were not included in the initial version.
- Advanced analytics and reporting features were limited to basic booking statistics.
